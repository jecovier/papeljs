Navigation Boost

By default, PapelJS fully renders both normal and partial pages. To enhance the navigation experience, you can use the pl-boost attribute on links to load only the partial content of a page.

With PapelJS performs an XHR call to fetch the partial page and loads its content into the appropriate slots, without reloading the entire page.

How It Works

  • When a link with the pl-boost attribute is clicked, PapelJS fetches the partial page content using an XHR request.
  • The fetched content is loaded directly into the designated slots on the current page.
  • This approach provides faster navigation by avoiding full page reloads.

Prefetching

To improve page loading times, you can use the pl-prefetch attribute on links. This attribute preloads the partial page into the browser's cache as soon as the link becomes visible in the viewport.

How It Works

  • pl-prefetch adds an event listener to the link.
  • When the link enters the viewport, PapelJS automatically fetches the partial page content and stores it in the browser's cache.
  • This ensures faster loading when the user clicks the link.

Prefetchin "ALL"

If you want to prefetch all the images in the next partial page, use the pl-prefetch="all" attribute. This will:

  • Preload the next partial page.
  • Fetch all images within that page.
  • Start loading those images into the browser's cache.
